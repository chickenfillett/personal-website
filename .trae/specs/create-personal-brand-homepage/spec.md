# Chicken Fillet 个人品牌主页 Spec

## Why

Chicken Fillet 作为一名独立软件开发者，需要建立一个个人品牌主页来展示其设计哲学与产品方向，同时为未来产品发布预留扩展空间。该主页还需满足支付平台（如 PayPal）对商家网站的审核要求，确保品牌具备商业可信度。

## What Changes

- 新建个人品牌主页网站，包含首页、关于页面、联系页面
- 首页包含品牌宣言、开发者理念、未来产品占位区
- 关于页面包含个人介绍、设计哲学、技术栈与联系方式
- 联系页面包含联系表单、邮箱、社交媒体链接
- 暗色极简视觉风格，统一设计系统
- 响应式布局，适配桌面与移动设备
- 预留产品展示扩展架构

## Impact

- 新增系统：个人品牌主页网站
- 受影响模块：无（全新项目）
- 关键文件：首页、关于页、联系页、全局样式、布局组件

## ADDED Requirements

### Requirement: 网站整体结构

The system SHALL 提供以下页面结构：

1. **首页（/）**：品牌入口，展示品牌宣言、设计哲学、未来产品预告
2. **关于页面（/about）**：个人介绍、开发理念、技术栈
3. **联系页面（/contact）**：联系表单、邮箱、社交媒体链接
4. **产品页面（/products）**：未来产品列表（当前为占位状态）
5. **404 页面**：极简风格的错误页面

#### Scenario: 导航访问

- **WHEN** 用户访问任意页面
- **THEN** 页面顶部显示固定导航栏，包含 Logo 和页面链接
- **AND** 导航栏在滚动时保持可见
- **AND** 当前页面链接高亮显示

### Requirement: 首页布局

The system SHALL 在首页按顺序展示以下模块：

1. **Hero 区域**：
   - 品牌名称 "Chicken Fillet"
   - 简短标语（如 "极简桌面工具的设计者"）
   - 暗色背景，无多余装饰

2. **理念宣言区域**：
   - 3-4 条核心设计哲学（极简、沉浸、克制、实用）
   - 简短文字，大留白

3. **产品预告区域**：
   - 标题 "即将推出"
   - 2-3 个产品占位卡片（显示产品名称与简短描述）
   - 卡片显示 "Coming Soon" 状态

4. **页脚**：
   - 版权信息
   - 快速链接
   - 社交媒体图标

#### Scenario: 首页加载

- **WHEN** 用户访问首页
- **THEN** 页面按上述顺序加载各模块
- **AND** 各模块间有充足留白
- **AND** 产品卡片显示为不可点击的占位状态

### Requirement: 关于页面

The system SHALL 在关于页面展示以下内容：

1. **个人介绍区**：
   - 开发者名称 "Chicken Fillet"
   - 身份定位：独立软件开发者
   - 简短个人故事（2-3 段）：为何选择独立开发、产品方向、设计哲学来源
   - 头像或抽象图形标识（可选）

2. **设计哲学区**：
   - 4 个核心原则，每项配简短说明：
     - **极简**：去除一切非必要元素
     - **沉浸**：让用户专注于任务本身
     - **克制**：功能不堆砌，每个特性都有存在的理由
     - **暗色**：减少视觉疲劳，适合长时间使用

3. **技术栈区**：
   - 简要列出主要技术方向（如桌面应用开发、UI 设计等）
   - 不使用复杂图标，纯文字列表

4. **行动号召区**：
   - 引导用户关注未来产品或联系开发者

#### Scenario: 关于页面访问

- **WHEN** 用户访问关于页面
- **THEN** 页面展示完整的个人介绍
- **AND** 设计哲学区使用卡片或列表形式展示
- **AND** 页面风格与首页保持一致

### Requirement: 视觉方向

The system SHALL 采用以下视觉设计规范：

1. **配色方案**：
   - 主背景色：深灰/近黑色（如 `#0a0a0a` 或 `#111111`）
   - 次背景色：稍浅深灰（如 `#1a1a1a`）
   - 主文字色：纯白或近白（如 `#ffffff` 或 `#f0f0f0`）
   - 次文字色：中灰（如 `#888888` 或 `#aaaaaa`）
   - 强调色：单一低饱和度色调（如暗蓝灰 `#4a5568` 或暗青 `#2d3748`），仅用于链接和按钮

2. **字体选择**：
   - 标题字体：无衬线字体（如 Inter、Helvetica Neue、system-ui）
   - 正文字体：与标题同族，保持统一
   - 字体层级：H1（品牌名，大字号）、H2（模块标题）、H3（卡片标题）、Body（正文）
   - 字重：标题使用 500-600，正文使用 400

3. **氛围与质感**：
   - 无渐变、无阴影、无圆角或极小圆角（2-4px）
   - 纯扁平设计，依靠色块区分层次
   - 大量留白，模块间距至少 80-120px
   - 无动画或仅极淡的过渡效果（0.2s ease）
   - 无装饰性图案、无插画、无 emoji

4. **布局原则**：
   - 最大内容宽度：1200px，居中
   - 移动端边距：16-24px
   - 桌面端边距：48-64px
   - 网格系统：简单单列或双列，不复杂

#### Scenario: 视觉一致性

- **WHEN** 用户浏览任意页面
- **THEN** 所有页面使用相同的配色、字体、间距
- **AND** 无亮色模式切换（仅暗色）
- **AND** 无视觉干扰元素

### Requirement: 支付平台审核要求

The system SHALL 满足支付平台（如 PayPal）对商家网站的基本要求：

1. **关于页面**包含：
   - 开发者/商家身份信息
   - 联系邮箱或联系表单

2. **联系页面**包含：
   - 有效的联系邮箱
   - 联系表单（姓名、邮箱、消息）

3. **页脚**包含：
   - 版权信息
   - 隐私政策链接（可链接到简单页面）

4. **整体网站**：
   - 专业外观，无占位符文本（Lorem Ipsum）
   - 所有链接可点击且指向有效页面
   - 网站可公开访问

#### Scenario: 审核检查

- **WHEN** 支付平台审核员访问网站
- **THEN** 能轻松找到关于页面和联系信息
- **AND** 网站呈现专业、可信的品牌形象
